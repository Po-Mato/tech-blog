# Contribution Guidelines - Root

This project adheres to **Conventional Commits** for commit messages.

## Commit Message Format

`<type>(<scope>): <subject>`

### Type
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation changes
- `style`: Code formatting/styling
- `refactor`: Code restructuring without behavior change
- `perf`: Performance improvements
- `test`: Adding/modifying tests
- `build`: Build system/tooling changes
- `ci`: CI/CD changes
- `chore`: Miscellaneous tasks
- `revert`: Reverting changes

### Scope (Recommended examples)
- `app`, `posts`, `components`, `universe`, `styles`, `deps`, `config`, `ci`

### Subject
- Written in Korean (concise, imperative mood), max 72 characters.
- No period at the end.

### Body (Optional)
- Explains *why*, *how*, and *impact* of the change.
- Max 72-100 characters per line.

### Footer (Optional)
- For linking issues/PRs (e.g., `Refs: #123`, `Fixes: #123`).
- For `BREAKING CHANGE` notifications.
