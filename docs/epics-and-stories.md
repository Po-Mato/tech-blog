# Epics & Stories — tech-blog

**작성일:** 2025-12-17

이 문서는 `tech-blog` 프로젝트의 구현을 위한 **에픽(Epic)** 및 **유저 스토리(User Story)**를 정의합니다.

- 기술 기준: Next.js(App Router) + React + TypeScript + Tailwind
- 현재 상태(코드 기반):
  - 홈(`app/page.tsx`)과 포스트 상세(`app/posts/[slug]/page.tsx`)는 placeholder
  - 배경 비주얼(`src/components/Universe.tsx`, `DynamicUniverse.tsx`)이 레이아웃에 포함됨

---

## 공통 정의

### 우선순위
- **P0**: 출시/기본 동작에 필수
- **P1**: 사용자 경험/운영에 중요
- **P2**: 있으면 좋음

### Definition of Done (DoD)
- 스토리별 Acceptance Criteria(AC)를 만족
- 로컬에서 `pnpm dev`로 정상 동작
- 빌드(`pnpm build`)가 성공
- 린트(`pnpm lint`)가 실패하지 않음
- 문서/설정 변경이 있으면 관련 문서 업데이트

---

## EPIC 1 — 블로그 정보구조(IA) & 네비게이션

### 목표
사용자가 홈에서 포스트 목록을 탐색하고, 카테고리/태그/검색 등으로 원하는 글을 빠르게 찾을 수 있게 한다.

### 범위(In)
- 홈에 포스트 리스트 노출
- 공통 네비게이션(상단 또는 사이드) 제공

### 비범위(Out)
- 로그인/회원 기능

### 성공 기준
- 홈에서 포스트 목록을 볼 수 있고, 각 항목에서 상세로 이동 가능

#### Stories

**S1.1 (P0) 홈에 포스트 리스트 렌더링**
- As a 사용자, I want 홈에서 최신 글 목록을 보고, 클릭해 상세 페이지로 이동하고 싶다.
- AC
  - 홈에 제목/요약/날짜/태그(선택)를 포함한 리스트가 보인다.
  - 각 항목 클릭 시 `/posts/[slug]`로 이동한다.
  - 포스트가 0개면 “아직 글이 없습니다” 빈 상태를 보여준다.

**S1.2 (P1) 공통 네비게이션 UI 추가**
- As a 사용자, I want 사이트 전체에서 홈/소개/카테고리로 쉽게 이동하고 싶다.
- AC
  - `app/layout.tsx` 기준으로 전 페이지에서 노출된다.
  - 모바일/데스크톱에서 레이아웃이 깨지지 않는다.

**S1.3 (P2) 태그/카테고리 페이지 제공**
- AC
  - 태그/카테고리별 필터 페이지가 존재한다.

## EPIC 2 — 포스트 콘텐츠 파이프라인(작성/저장/렌더링)

### 목표
포스트가 실제 콘텐츠(예: Markdown/MDX)로 관리되고, 상세 페이지에서 올바르게 렌더링되도록 한다.

### 범위(In)
- 콘텐츠 저장 위치/형식 결정
- slug 기반 라우팅 및 정적 생성(가능하면)

### 성공 기준
- 포스트가 placeholder가 아니라 실제 콘텐츠로 렌더링된다.

#### Stories

**S2.1 (P0) 포스트 소스 형식 및 디렉터리 결정**
- AC
  - 예: `content/posts/*.md(x)` 같은 형태로 저장 규칙이 문서화된다.
  - 각 포스트는 최소한 `title`, `date`, `slug` 메타데이터를 가진다.

**S2.2 (P0) `/posts/[slug]`에서 실제 콘텐츠 렌더링**
- AC
  - slug로 포스트를 찾아 렌더링한다.
  - slug가 없으면 404 처리한다.
  - 코드 블록/링크/이미지 렌더링이 깨지지 않는다(기본 수준).

**S2.3 (P1) 포스트 목록 데이터 소스 연결**
- AC
  - 홈 리스트가 S2.1에서 정한 데이터 소스에 의해 생성된다.

## EPIC 3 — 디자인/브랜딩 & 배경(유니버스) 통합

### 목표
가독성 좋은 블로그 UI를 제공하면서 배경 비주얼(유니버스)을 방해 요소가 아닌 브랜드 요소로 통합한다.

### 범위(In)
- 배경 레이어링/대비/접근성
- 성능(프레임, 리소스 사용량) 고려

### 성공 기준
- 텍스트 가독성이 유지되고, 스크롤/리사이즈 시 문제 없다.

#### Stories

**S3.1 (P0) 배경 위 콘텐츠 가독성 보장**
- AC
  - 본문 영역에 반투명 배경/블러 등으로 대비를 확보한다.
  - 텍스트 대비가 충분하다(최소 수준의 접근성 고려).

**S3.2 (P1) 배경 애니메이션 성능 최적화**
- AC
  - 탭 비활성화/페이지 전환/언마운트 시 리소스가 해제된다.
  - 리사이즈 핸들러가 과도하게 호출되지 않는다(필요 시 디바운스).

**S3.3 (P2) 사용자 설정: 배경 효과 끄기**
- AC
  - “배경 끄기” 토글이 제공되고 설정이 유지된다(로컬스토리지 등).

## EPIC 4 — SEO/메타데이터/사이트맵

### 목표
검색엔진 및 공유(OG)에서 기본 정보가 올바르게 노출되도록 한다.

#### Stories

**S4.1 (P0) 사이트 기본 메타데이터 정리**
- AC
  - `app/layout.tsx`의 `metadata`가 실제 블로그 정보로 채워진다(제목/설명).

**S4.2 (P1) 포스트별 메타데이터/OG 적용**
- AC
  - 포스트 제목/요약이 OG에 반영된다.

**S4.3 (P1) sitemap/robots 제공**
- AC
  - 최소한의 사이트맵이 생성/서빙된다.

---

## EPIC 5 — 운영/품질(분석/에러/배포)

### 목표
배포 이후에도 사이트 상태를 파악하고 안정적으로 운영할 수 있게 한다.

#### Stories

**S5.1 (P0) 배포 환경에서 빌드/런타임 확인**
- AC
  - 배포 워크플로가 현재 앱 구조와 일치한다.
  - 배포 후 홈/상세 페이지가 정상 동작한다.

**S5.2 (P1) 방문 분석 도입(선택)**
- AC
  - 개인정보/쿠키 정책에 맞는 방식으로 최소한의 트래킹을 적용한다.

**S5.3 (P1) 에러 로깅/관측성(선택)**
- AC
  - 사용자 에러를 수집할 수 있다(예: Sentry 등).
